    로그인 / 홈(역할별 랜딩)

    TL;DR

    • L-01 로그인: 이메일/비밀번호 최소 UX, 에러 카피 정의
    • H-01 My Home: GET /me 기반 역할별 랜딩 라우팅 (과투자 방지)

    🔐 1. L-01 로그인 화면 (UX 최소 스펙)

    화면 구성

    요소 내용
    제목 로고 + "로그인" 텍스트
    입력 필드 이메일 (type=email) / 비밀번호 (type=password)
    액션 버튼 "로그인" (primary)
    로그인 유지 "로그인 상태 유지" (Remember Me) 체크박스
    부가 링크 "비밀번호를 잊으셨나요?" (선택, Phase 1)

    에러 카피 (서버 응답 기준)

    상황 UI 메시지
    이메일 형식 오류 "올바른 이메일 형식을 입력해주세요."
    이메일/비밀번호 불일치 "이메일 또는 비밀번호가 올바르지 않습니다."
    비활성화 계정 "비활성화된 계정입니다. 관리자에게 문의하세요."
    서버 오류 "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."

    Phase 1: OTP/소셜 로그인은 제외. 이메일+비밀번호만.

    🏠 2. H-01 My Home (역할별 랜딩 라우팅)

    목적

    GET /me 응답의 role 값을 기준으로 로그인 직후 적절한 화면으로 자동 이동합니다. H-01 자체는 별도 UI가 아닌 라우팅 레이어 역할입니다.

    역할별 랜딩

    role 랜딩 화면 설명
    ADMIN A-00 (Admin 대시보드) 팀별 운영지표 MVP 카드
    TM T-00 (팀 대시보드) 기사별 운영지표 MVP 카드
    TECH M-01 (기사 작업 목록) 본인 배정 작업 목록

    ⏳ Loading State (스켈레톤 UI)

    GET /me 호출 및 라우팅 판단 중에는 사용자에게 스켈레톤 UI(Skeleton UI) 또는 로딩 상태를 노출하여 앱이 멈춘 것이 아님을 인지시킵니다. (Phase 1 프리미엄 UX)

    세션 만료/토큰 무효 시 → L-01 로그인 화면으로 리다이렉트. 과투자 방지: H-01은 별도 홈 피드/알림 피드 없음. 최소 라우팅만.

    � 3. 세션 관리 & 보안 정책

    세션 만료 경고 및 연장

    • 만료 예고: 토큰 만료 5분 전, 상단 혹은 모달로 "세션이 곧 만료됩니다" 알림 노출.
    • 세션 연장: 알림 내 "연장하기" 버튼 클릭 시 Refresh Token을 사용하여 세션을 즉시 갱신.
    • 자동 로그아웃: 갱신 없이 만료 시, 현재 작업 중인 데이터를 보호하기 위해 안내 메시지 노출 후 L-01로 리다이렉트.

    잘못된 접근(Unauthorized) 처리

    • 권한이 없는 페이지(예: TECH가 ADMIN 설정 진입) 직접 접속 시도 시:
    • 접근 차단 메시지(Toast) 노출
    • H-01(역할별 랜딩 홈)로 자동 리다이렉트

    �🔗 3. GET /me Endpoint 정의

    항목 내용
    Method GET
    Path /api/v1/me
    Auth Bearer Token 필수
    Role ALL (ADMIN / TM / TECH)
    Scope SELF

    응답 예시:

    {
  "ok": true,
  "data": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "홍길동",
    "role": "TM",
    "team_id": "uuid",
    "team_name": "서울 북부 센터",
    "status": "ACTIVE"
  }
}

    status: INACTIVE인 경우 서버가 401을 반환하여 로그인 차단.
