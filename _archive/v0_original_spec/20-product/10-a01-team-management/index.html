    A-01 팀 관리 + A-00 Admin 대시보드

    TL;DR

    • A-01: Admin이 팀(센터)을 생성·수정·비활성화하는 관리 화면. 삭제 금지, 팀명 Immutable.
    • A-00: Admin 홈 대시보드. 팀별 운영지표 MVP 카드. 성과평가 아님.

    🏢 1. A-01 팀 관리

    1.1 팀 목록 화면

    요소 내용
    목록 표시 팀명, 상태(ACTIVE/INACTIVE), 소속 기사 수, 생성일
    필터 상태 필터 (전체 / ACTIVE / INACTIVE)
    액션 [팀 생성] 버튼, 각 행 클릭 → 팀 상세

    1.2 팀 생성 폼

    필드 필수 비고
    팀명(센터명) ✅ 필수 생성 후 변경 불가 (Immutable)
    센터 주소 선택
    센터 연락처 선택
    담당자 이름 선택

    안내 카피: "팀명은 한번 등록하면 변경할 수 없습니다. 신중하게 입력해주세요."

    1.3 팀 상세/수정 화면

    필드 수정 가능 여부
    팀명(센터명) ❌ 불가 (읽기 전용 표시)
    센터 주소 ✅ 가능
    센터 연락처 ✅ 가능
    담당자 이름 ✅ 가능

    1.4 팀 비활성화

    항목 내용
    액션 "팀 비활성화" 버튼 (상세 화면 하단)
    사유 입력 창 제공 (선택 사항)
    확인 모달 "비활성화된 팀의 기사는 로그인 및 신규 배정이 차단됩니다."
    삭제 금지 삭제 버튼 없음 (Phase 1)

    📊 2. A-00 Admin 대시보드 (운영지표 MVP)

    2.1 카드 구성

    카드 지표 정의
    팀별 상태 카운트 DRAFT / TEAM_ASSIGNED / IN_PROGRESS / COMPLETED 해당 팀에 배정된 WO 상태별 건수
    지연 카운트 scheduled_at < now() AND status != COMPLETED 예정일 지난 미완료 건수

    2.2 인터랙션

    동작 결과
    팀 카드 클릭 A-02(본사 작업 목록)으로 이동, 해당 팀 필터 자동 적용
    지연 카운트 클릭 A-02로 이동, "지연" 필터 자동 적용

    통계는 "운영지표(MVP)" 의미입니다. 개인 성과평가 용도로 사용하지 않습니다.

    2.3 화면 구성

    요소 내용
    헤더 "운영 현황 (오늘 기준)" + 마지막 갱신 시간
    팀 카드 그리드 팀명, 상태별 카운트, 지연 카운트
    상단 [팀 관리] 링크 A-01로 이동

    연관 문서

    • 권한 매트릭스(RBAC)
    • Admin 작업 목록 (A-02)
    • API 최소세트
