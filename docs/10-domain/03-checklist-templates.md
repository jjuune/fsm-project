# 03 체크리스트 및 점검 템플릿 표준

## TL;DR
- **목적:** 현장 설치 품질의 표준화 및 데이터 기반의 증빙 체계 구축
- **운용 방식:** Phase 1에서는 **'표준 설치 템플릿'** 하나를 모든 작업에 공통 적용하며, 향후 제품별 특화 템플릿으로 확장 가능한 구조를 지향함

---

## 1. 체크리스트 항목 생성 원칙 (Standard)

모든 체크리스트 항목은 다음 5가지 기준을 바탕으로 설계하며, 기사님이 현장에서 오판하지 않도록 명확한 문구로 작성합니다.

1. **설치 전 환경 점검:** 주변 영향 요소(누수, 과열, 공간 확보 등) 사전 확인
2. **설치 품질 확인:** 제품의 수평/수직, 고정 상태, 외관 흠집 등 물리적 상태 점검
3. **기능 정상 작동:** 전원, 소음, 기초 동작 등 즉각적인 성능 확인
4. **안정성 모니터링:** 일정 시간 가동 후 수치 변화(수위, 온도 등) 기록
5. **고객 케어:** 주요 기능 설명 및 주의사항 대면 안내 완료 여부

---

## 2. Phase 1 표준 설치 점검 템플릿 (MVP)

WorkOrder 생성 시 다음 항목들이 자동으로 생성되어 기사님 앱(M-02)에 노출됩니다.

| 순번 | 점검 항목 | 입력 타입 | 필수 | 비고 |
| :---: | :--- | :---: | :---: | :--- |
| 1 | 설치 장소 주변의 누수 또는 과열 위험 요소가 없는가? | PASS/FAIL | ✅ | FAIL 시 사유 입력 |
| 2 | 제품이 바닥면에 **평탄도**에 맞게 안정적으로 설치되었는가? | PASS/FAIL | ✅ | 수평 유지 확인 |
| 3 | 전원 연결 시 초기 구동음 및 동작 상태가 정상인가? | PASS/FAIL | ✅ | 이상 소음 확인 |
| 4 | [성능] 10분간 연속 가동 후 목표 수위/수치에 도달했는가? | PASS/FAIL | ✅ | 수치 기록 권장 |
| 5 | 가동 중 연결 부위에서 미세 누수나 이상 발열이 없는가? | PASS/FAIL | ✅ | 육안/촉각 점검 |
| 6 | 고객에게 제품 사용법 및 긴급 시 전원 차단법을 설명했는가? | PASS/FAIL | ✅ | 대면 안내 |
| 7 | 기존 설치 환경 복구 및 주변 정리가 완료되었는가? | PASS/FAIL | ✅ | 현장 정리 정돈 |

---

## 3. 예외 및 특이사항 정책

- **FAIL 처리 시:** 기사님은 반드시 **'메모(Note)'**를 입력해야만 완료 제출이 가능함 (시스템 강제)
- **수치 입력 (확장):** 필요 시 특정 항목에 대해 텍스트나 숫자를 직접 입력할 수 있는 필드를 제공함

---

## 🚀 미래 확장성 (Phase 1.x / Gate 2)

- **제품별 템플릿 매핑:** 현재는 단일 표준 템플릿을 사용하나, 향후 **Admin -> 제품 관리** 기능과 연계하여 제품 카테고리별로 각기 다른 체크리스트 템플릿이 자동으로 호출되도록 시스템을 확장할 예정임.
- **사진 연동:** 특정 항목(예: 평탄도 확인)에 대해 특정 사진 촬영을 강제하는 '포토 체크리스트' 기능 검토.
