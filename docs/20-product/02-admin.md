# A-01 Admin(본사 관리자) 화면 상세 스펙

## TL;DR
- **목적:** 조직 전체 운영 지표 상시 모니터링 및 미배정 작업의 즉각적인 팀 할당(Assignment)
- **핵심 화면 구성:**
  - **A-00 Admin 대시보드:** 요약 지표 (신규 추가 필요)
  - **A-01 팀 관리:** 마스터 관리
  - **A-02 작업 목록:** 운영 대응 (팀 배정 중심)
  - **A-04 작업 상세:** 컨트롤 타워 (이력 확인 및 예외 처리)

---

## 📊 A-00 Admin 대시보드 (신규)
본사 관리자가 로그인 후 가장 먼저 마주하는 화면으로, 조직 전체의 건강 상태를 확인합니다.
- **주요 요소:**
  - **상태별 카운트:** 총 작업 수, 진행 중, 지연 건수, 오늘 완료 건수
  - **팀별 운영 현황:** 각 팀별 작업 처리 속도 및 미배정 건수 요약 차트/리스트
  - **긴급 이슈 알림:** 지연(Overdue) 시간이 임계치를 넘은 작업 목록 노출

---

## 📋 A-02 작업 목록 (팀 배정 중심)
미배정 작업을 확인하고 즉시 팀을 배정하는 운영의 핵심 화면입니다.

### 1. 주요 기능 및 UI 요소
| 요소 | 상세 내용 | 비고 |
| :--- | :--- | :--- |
| **KPI 필터 카드** | 팀 미배정 / 오늘 예정 / 지연 / 오늘 완료 | 클릭 시 목록 즉시 필터링 |
| **퀵 팀 배정** | 목록 우측 [팀 배정] 버튼 노출 | 배정되지 않은 건에만 노출 |
| **일괄 작업** | 여러 건 선택 후 동일 팀에 일괄 배정 | (Phase 1.5 권장) |

### 2. 퀵 팀 배정 플로우
1. 목록에서 [팀 배정] 클릭 ➔ 2. 팀 선택 모달(드롭다운) 오픈 ➔ 3. 팀 선택 및 메모 입력 ➔ 4. [확인] 클릭 ➔ 5. 상태가 `TEAM_ASSIGNED`로 변경되며 목록 갱신

---

## 🔍 A-04 작업 상세 (컨트롤 타워)
개별 작업의 모든 생애주기(Lifecycle)를 추적하고 제어합니다.

### 1. 섹션 구성
- **Header:** 작업 번호, 현재 상태 배지, [작업 취소] 버튼 (상태에 따라 활성)
- **Info Card:** 고객 정보(성함/연락처), 현장 주소(지도 링크), 요청 사항
- **Assignment Zone:** 담당 팀 및 담당 기사 정보 (변경 필요 시 사유 입력 후 변경 가능)
- **Evidence Panel:** 입력된 체크리스트(✅/⚠️), 고객 서명 이미지, 첨부 사진 갤러리
- **Timeline (Audit Log):** 생성 ➔ 팀 배정 ➔ 기사 배정 ➔ 시작 ➔ 완료의 전 과정을 시간순 노출

### 2. 주요 정책 (단호박 👩‍🏫)
- **취소 정책:** `IN_PROGRESS`까지는 취소 가능하나, `COMPLETED` 이후에는 원칙적 취소 불가.
- **재발송 기능:** 고객이 보고서를 못 받았다고 할 경우, PDF 보고서 수동 재발송(SMS/Email) 기능 제공.

---

## 🛠️ A-01 팀 관리 (센터 마STER)
본사 하위의 운영 조직(팀/센터)을 관리합니다.
- **필드:** 팀명(Immutable), 주소, 연락처, 팀장 명, 상태(Active/Inactive)
- **규칙:** 삭제 대신 **비활성화(Deactivate)** 처리하여 과거 이력 보존.

---

## 🚀 확장 (Gate 2: 상품 카탈로그)
- **A-05 상품 분류 관리:** 서비스 대상 제품군(예: 공기살균기, 비데 등) 분류 관리
- **A-06 상품 상세 관리:** 모델명, SKU, 대표 이미지 등 마스터 정보 관리
