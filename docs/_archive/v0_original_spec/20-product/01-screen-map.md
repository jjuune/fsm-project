# 화면 지도(Screen Map)

## TL;DR
- 역할별 화면 구성과 상세 설계 문서를 한눈에 파악하는 **프로젝트 내비게이션**입니다.
- 초기 원고(.docx)의 흔적을 지우고, 레포지토리 내의 **최신 마크다운(.md) 문서**를 기준으로 관리합니다.

## 🛠️ 화면맵 관리 규칙
1. **Source of Truth**: 모든 화면의 상세 정의는 본 문서의 링크를 따라가는 **`.md` 파일**이 우선합니다.
2. **독립 화면 기준**: 새로운 URL이나 독립적인 레이아웃이 필요한 경우에만 새 ID를 부여합니다.
3. **확장 기능(Gate 2)**: 현재 스코프 밖의 기능은 `[Gate 2]` 태그를 붙여 관리하고, UI 확장은 bullet 포인트로 명시합니다.
